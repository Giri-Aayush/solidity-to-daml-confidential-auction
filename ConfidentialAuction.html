watch the privacy hold

place a sealed bid. see exactly what each party is, and isn't, allowed to read.

Four parties, four ledgers. The auctioneer co-signs every bid, so it sees all of them. Each bidder signs only their own. Flip to EVM and every redaction bar drops at once: the difference between a private ledger and a public one, in a click.

unrevealed bids are forfeited at settlement

Redaction bars are a teaching aid. On a live Canton deployment a sealed bid never reaches a non-stakeholder's participant node at all, so they wouldn't see even a placeholder.

the same privacy, in code

on the evm, privacy is machinery. on canton, it's the type.

The Solidity contract spends commit, reveal, deposits, and a timeline to fake a secret. The Daml template just names who is allowed to see the bid, and the rest disappears.

// hide the bid behind a hash + a deposit
function commit(bytes32 blindedBid) external payable {
    if (blindedBid == bytes32(0)) revert EmptyCommitment();
    bids[msg.sender] = Bid(blindedBid, msg.value, false);
}

// reveal it later — now everyone can see it, forever
function reveal(uint256 value, bytes32 secret) external {
    Bid storage bid = bids[msg.sender];
    bool valid = bid.blindedBid == _commitmentOf(value, secret)
        && bid.deposit >= value && value > 0;
    if (valid && value > highestBid) {
        highestBid = value; highestBidder = msg.sender;
    }
}

the reveal phase

No hash to commit to, so no second window to reveal it. A bid is born private.

deleted

deposits & forfeiture

These existed only to make a hidden commitment binding. A Daml bid is a real, authorized contract the instant it is created.

deleted

the reentrancy surface

No value escrowed mid-contract and no external calls during execution, so the pull-payment dance isn't needed.
